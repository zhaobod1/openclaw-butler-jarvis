# ADR-001: ClawX Fork 品牌定制

**日期**: 2026-07-09
**状态**: 已实施
**决策者**: job zhao

## 背景

需要为中国用户提供 OpenClaw 的桌面客户端，满足：
1. 品牌为中国本土化（龙虾管家）
2. 开箱即用（预配置聚星逸 AI Provider）
3. 便携分发（USB 安装包）

## 方案对比

### 方案 A: 从零自研 Electron 客户端（已否决）
- 工作量巨大（ClawX ~700 文件、数百组件、完整 Gateway 集成）
- 需要独立维护 OpenClaw 集成逻辑
- 没有复用社区生态

### 方案 B: Fork ClawX 品牌定制（已采纳 ✅）
- 直接复用 ClawX 的完整 Electron 架构
- 只需改动品牌层（i18n、图标、配置）
- 可定期同步上游更新
- 维护成本低

### 方案 C: 使用 OpenCami（Web 版，已否决）
- Web 客户端功能弱于桌面
- 无系统托盘、本地文件集成

## 决策

采纳方案 B，fork ClawX v0.4.15 进行品牌定制。

## 影响

- **正面**: 快速交付、完整功能、可同步上游
- **负面**: 依赖上游架构演进，品牌改动需持续维护
- **约束**: 不推送到上游，只维护私仓

## 具体改动范围

见 `docs/developer-sop.md` § 文件变迁清单。

## 被否方案

- 自研客户端：需 3-6 个月开发周期
- OpenCami 二次开发：缺少桌面原生能力
- 仅 CLI 方案：用户需要 GUI
